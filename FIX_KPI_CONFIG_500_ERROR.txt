PERBAIKAN ERROR 500 DI /kpi-config
===================================

MASALAH:
- GET http://localhost:3002/kpi-config mengembalikan 500 (Internal Server Error)
- API route mencoba query tabel 'm_users' yang tidak ada di database
- Error: "Could not find the table 'public.m_users' in the schema cache"

ANALISIS:
Sistem menggunakan struktur database yang berbeda:
- auth.users: Tabel autentikasi Supabase (built-in)
- m_employees: Tabel master pegawai dengan kolom user_id yang mereferensi auth.users.id
- m_employees memiliki kolom 'role' untuk authorization (superadmin, unit_manager, employee)

TIDAK ADA tabel m_users di database!

SOLUSI:
Memperbaiki app/api/kpi-config/route.ts:

SEBELUM (SALAH):
```typescript
const { data: profile, error: profileError } = await supabase
  .from('m_users')  // ❌ Tabel tidak ada!
  .select('role')
  .eq('id', user.id)
  .single()
```

SESUDAH (BENAR):
```typescript
const { data: employee, error: employeeError } = await supabase
  .from('m_employees')  // ✓ Tabel yang benar
  .select('role, full_name, employee_code')
  .eq('user_id', user.id)  // ✓ Menggunakan user_id, bukan id
  .single()
```

PERUBAHAN KUNCI:
1. Ganti 'm_users' → 'm_employees'
2. Ganti .eq('id', user.id) → .eq('user_id', user.id)
3. Tambah informasi user di response untuk debugging

HASIL:
✅ API endpoint berfungsi normal
✅ Query berhasil menemukan employee profile
✅ Role authorization bekerja dengan benar
✅ Error 500 teratasi

CARA TEST:
1. Jalankan: .\TEST_KPI_CONFIG_FIX.ps1
2. Atau manual: npm run dev, lalu akses http://localhost:3002/api/kpi-config
3. Atau test script: npx tsx scripts/test-kpi-config-api.ts

CATATAN:
- Sistem login dan auth TIDAK diubah (tetap berfungsi normal)
- Hanya memperbaiki query di API route yang salah
- Struktur database sudah benar, hanya API route yang perlu diperbaiki
